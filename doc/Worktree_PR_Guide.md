# 워크트리 작업 및 병합 가이드

에이전트에게 워크트리를 사용하여 작업을 지시하고, 완료된 작업을 로컬에서 직접 병합하거나 PR로 검토하는 방법입니다.

## 1. 에이전트에게 작업 지시하기 (Worktree 활용)

에이전트에게 **"/worktree 워크플로우를 사용하여 [작업 내용]을 처리해줘"**라고 요청합니다.

### 예시 명령어:
> "/worktree 워크플로우를 사용해서 Quaternion 회전 로직의 버그를 수정해줘. 브랜치명은 `fix-quat`으로 해줘."

### 에이전트가 수행하는 단계:
1.  `./worktree-create.sh fix-quat` 실행 (로컬에 전용 작업 공간 생성).
2.  해당 워크트리 폴더로 이동하여 코드 수정 및 테스트.
3.  작업 완료 알림.

## 2. 로컬에서 직접 병합하기 (PR 없이)

리모트 PR을 거치지 않고 로컬에서 즉시 메인 브랜치에 반영하는 방법입니다.

1.  **메인 브랜치에서 에이전트에게 병합 지시** (가장 권장):
    > "`fix-quat` 브랜치의 작업을 로컬 `main` 브랜치에 병합하고 워크트리를 삭제해줘."
2.  **직접 명령어로 병합**:
    ```bash
    git merge wt-fix-quat
    ```
    - 에이전트가 만든 브랜치는 기본적으로 `wt-[작업명]` 형식을 갖습니다.

## 3. PR(Pull Request) 활용하기

협업 중이거나 원격 저장소를 통한 검토가 필요할 때 사용합니다.

- **명령**: "/worktree 작업을 완료한 뒤 리모트에 푸시하고 PR을 생성해줘."
- **수락**: GitHub/GitLab 웹 UI 또는 CLI(gh command 등)를 통해 병합.

## 4. 작업 정리

병합이 완료되면 시스템을 깨끗하게 유지하기 위해 워크트리를 삭제합니다.
- **명령**: "`fix-quat` 워크트리 삭제해줘."
- **직접 실행**: `./worktree-delete.sh fix-quat`

---
> [!TIP]
> 로컬 병합 시 충돌이 발생하면 에이전트에게 "병합 충돌을 해결해줘"라고 지시할 수 있습니다.

---
> [!TIP]
> 에이전트는 `/worktree` 슬래시 커맨드를 통해 정의된 절차를 인식하므로, 명령 시 이 키워드를 포함하는 것이 가장 정확합니다.
