# CharacterController 벽 경계 초과 슬라이딩 수정

## 유저 보고 내용
- static BoxCollider 벽에서 슬라이딩 시작 후, 키를 계속 누르고 있으면 벽 경계를 넘어서도 허공에서 같은 방향으로 슬라이딩이 계속됨
- 키를 떼었다 다시 누르면 정상 동작

## 원인
`_prevContactNormal` 리셋 로직의 결함.

이전 jitter 수정(fix-cc-wall-jitter-v2)에서 추가한 프레임 간 벽 법선 보존 시스템에서, 현재 프레임에서 벽 충돌이 없을 때의 처리가 문제:
- 원래 motion이 벽을 향하고 있으면(`dotOriginal < -0.001f`) 법선을 **무조건 유지**
- 이 조건은 "사전 제거가 잘 작동해서 충돌이 안 된 것"이라고 가정하지만, 실제로는 "벽 경계를 넘어서 벽이 없어서 충돌이 안 된 것"일 수도 있음
- 벽을 향해 키를 계속 누르고 있으면 motion의 벽 방향 dot이 항상 음수이므로, 벽이 없어도 법선이 영원히 유지됨
- 결과: 허공에서 벽 방향 성분이 계속 사전 제거되어 슬라이딩이 지속됨

## 수정 내용
벽 충돌이 없을 때 법선을 무조건 유지하는 대신, **벽이 실제로 존재하는지 probe sweep**으로 확인:
- 보존된 법선의 반대 방향(벽을 향해)으로 `skinWidth * 3` 거리의 짧은 sweep 수행
- 벽이 감지되면 법선 유지 (jitter 방지를 위해 필요)
- 벽이 감지되지 않으면 법선 클리어 (벽 경계를 넘었으므로 더 이상 슬라이딩 불필요)

이 방식은 기존 jitter 방지 기능을 유지하면서, 벽이 없는 허공에서의 잘못된 슬라이딩을 정확히 해결한다.

## 변경된 파일
