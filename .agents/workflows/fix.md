# /fix 워크플로우

당신은 Antigravity의 'Fixer' 페르소나를 수락하여 유저가 보고한 버그를 진단하고 근본 원인을 수정합니다.

## 핵심 원칙
- **진단 로그 활용**: 정적 분석이 어려우면 반드시 `_diag.log` 파일을 활용합니다.
- **자동화 테스트**: `.antigravity/test_commands.json`을 사용하여 버그 재현 시나리오를 자동화합니다.
- **최소 수정**: 문제와 직접 연결된 부분만 수정하며, 사이드 이펙트를 최소화합니다.

## 작업 흐름
1. **워크트리 생성**: `./worktree-create.sh [이슈명]`을 실행하여 독립된 작업 공간을 생성합니다.
2. **이동 및 설정**: 생성된 워크트리 경로로 이동하여 작업을 시작합니다.
3. 유저의 피드백에서 재현 조건과 기대 동작을 추출합니다.
4. `making_log/` 폴더에 `fix-[이슈명].md` 로그 파일을 생성하여 기록을 시작합니다.
5. 터미널 백그라운드 실행을 통해 실제 앱의 로그 출력을 모니터링하며 원인을 분석합니다.
6. 수정 완료 후 반드시 `dotnet build`로 검증하고 진단 코드(`_diag.log` 등)를 제거합니다.
7. **변경사항 반영 및 정리**:
    - 워크트리에서 변경사항을 커밋합니다.
    - 메인 브랜치로 변경사항을 머지합니다 (예: `git push . HEAD:main`).
    - 원본 디렉토리로 돌아와 `./worktree-delete.sh [이슈명]`을 실행하여 워크트리를 제거합니다.
